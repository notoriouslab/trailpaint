## 這是什麼 / What is TrailPaint

TrailPaint 是一個純瀏覽器端的手繪風路線地圖工具，適合登山、單車、旅遊部落客、生態導覽、教育單位、教會 NGO 等使用。所有資料存在瀏覽器本地，不上傳伺服器；支援 PWA 可安裝到手機主畫面像原生 App。

TrailPaint is a browser-only hand-drawn style trail map maker, designed for hikers, cyclists, travel bloggers, ecology guides, educators, and churches/NGOs. All data is stored locally in the browser; PWA-installable on phones, tablets, and desktops.

## 三種建立路線的方式 / Three Ways to Build a Trail

1. **從照片建立（v1.3 新能力）**：拖 20 張照片 → EXIF GPS 自動建景點。支援 iPhone HEIC、Android JPEG。無 GPS 照片歸入「待定位」分組。Nominatim 反向地理編碼自動補地名。
   _From Photos (new in v1.3): drag up to 20 photos; EXIF GPS auto-creates spots. iPhone HEIC and Android JPEG supported._

2. **在地圖上手動建**：搜尋地點、畫路線、放景點卡片。3 步引導教學 + 8 條範例路線試玩。
   _Manually on the Map: search locations, draw routes, place spot cards. 3-step tutorial, 8 sample trails._

3. **用 AI 生成**：複製內建提示詞給 ChatGPT / Claude，描述行程 → 產可匯入的 JSON。
   _Generated by AI: copy prompt template to ChatGPT / Claude; describe your trip; get importable JSON._

## 匯入格式 / Import Formats

- `.trailpaint` project file (native)
- Photo EXIF (JPEG, HEIC with GPS)
- KML (Google My Maps, geojson.io)
- GeoJSON (Google Takeout, Mapbox, D3)
- GPX (hiking apps, Garmin)
- Screenshot basemap (drag any image)

## 輸出格式 / Output Formats (unified ExportWizard v1.3)

Three tabs for three use cases:

### Image tab
- PNG at multiple ratios: 1:1 (IG feed), 9:16 (Story), 4:3, original
- 3 border styles: Classic double, Paper sketch, Minimal thin
- 2 style filters: Original, Sketch
- Share link via Cloudflare Workers + KV short URL (TTL 90 days, Open Graph cover from first photo)
- AI prompts in 4 styles: Japanese hand-drawn, Treasure map, Kawaii cartoon, Minimal line art
- Player embed code (`<iframe>` with project data baked in)

### Backup tab
- `.trailpaint` project file (preserves all spots, routes, photos)
- Import `.trailpaint` bridge to ImportWizard

### Interop tab
- GeoJSON (for geojson.io, Mapbox, D3 visualizations)
- KML (for Google My Maps, Google Earth, Gaia GPS)
- Pure geographic structure, no photos (open data boundary)

## Privacy and Data

- **Zero backend**: all project data stored in browser localStorage, never uploaded
- **Zero account**: no signup required
- External calls only: map tiles (OSM / CARTO / Protomaps), Photon (primary) / Nominatim (fallback) reverse geocoding, Open-Meteo elevation, Cloudflare Workers share-link backend (user-triggered)

## Tech Stack

- Framework: Vite + React 19 + TypeScript 5 (strict)
- Map: Leaflet + react-leaflet + protomaps-leaflet
- State: Zustand + zundo (temporal undo/redo)
- PWA: vite-plugin-pwa + Workbox (active update prompt)
- Import: exifr (EXIF GPS incl. HEIC), @tmcw/togeojson (KML), custom GeoJSON parser
- Export: html-to-image + Canvas post-processing + style filters
- Multi-page build: Editor at /app/, Player at /app/player/
- Story Page: static HTML + vanilla JS (native SEO/OG)
