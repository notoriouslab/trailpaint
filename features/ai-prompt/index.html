AI 提示詞生成地圖

把景點描述 / 旅遊計畫 / 宣教路線丟給 LLM，產 .trailpaint.json，匯入即成地圖。Editor 內建 4 種風格提示詞。

使用流程

  1. 打開 Editor → 右下「匯出」按鈕 → 圖片 tab
  2. 下拉選「AI 提示詞風格」（登山 / 宣教 / 歷史 / 旅遊四種）
  3. 點「複製 AI 提示詞」
  4. 貼到 ChatGPT / Claude / Gemini，填進你的主題（例：「我想畫阿朗壹古道 5 個景點」）
  5. LLM 會回傳一段 .trailpaint.json 內容
  6. 存成 .trailpaint.json 檔，從 Editor 的「匯入」→「.trailpaint 專案檔」載入

四種內建風格

登山 Hiking

輸出著重山區景點：三角點、山屋、水源、危險地形、展望點。路線帶海拔起伏描述。圖示偏向 ⛰️ 🏕️ 💧。

宣教 Mission

輸出著重宣教士足跡：教會、醫館、學校、原住民部落、殉道地。每景點含年份、人物、歷史事件。圖示偏向 ⛪ 🏥 🏫。

歷史 History

輸出著重歷史遺跡：古戰場、考古地點、古城門、舊碼頭。每景點含朝代 / 年代、考證參考。圖示偏向 🏛️ ⚔️ 📜。

旅遊 Travel

輸出著重觀光體驗：景點、美食、住宿、交通樞紐。每景點含推薦停留時間、預算。圖示偏向 📸 🍜 ☕ 🏨。

範例：用 Claude 產「台北一日行程」

提示詞（從 Editor 複製的）

你是路線地圖 JSON 生成器。請為主題「{TOPIC}」產一份
TrailPaint .trailpaint.json 格式的路線地圖，包含：
- version: 3
- name: 路線名稱
- center: [lat, lng] 地圖中心
- zoom: 13-15
- spots: 5-8 個景點，每個包含 id/latlng/num/title/desc/iconId
- routes: 1-2 條路線，pts: [[lat,lng], ...]
- iconId 從 31 種選：home/church/camera/food/coffee/peak/...
直接回 JSON，不加任何說明文字。

注意事項

  • 座標準確性：LLM 對常見地標座標基本準確，冷門地點可能漂移 100-500 公尺，建議匯入後手動微調
  • 照片：AI 不會產真實照片 data URL；景點會無照片，需之後手動補
  • schema 版本：請要求 LLM 用 v3；舊版本 TrailPaint 會自動 migrate
  • 免費 LLM：ChatGPT Free、Claude.ai Free、Gemini 都能用；Claude 對結構化 JSON 輸出最穩定

進階：給 ChatGPT 照片直接 EXIF 生成

把手機相簿選 10 張照片交給 ChatGPT-4V 並說「從這些照片 EXIF 產 .trailpaint.json」— ChatGPT 能讀 EXIF GPS 直接產景點。但 TrailPaint 本身就有「匯入照片」功能（更快也更準），這個 workflow 主要適合「照片在別人手上、你只拿到檔案」的協作情境。
