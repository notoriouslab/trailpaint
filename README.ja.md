## すぐに使う

### オンライン版（推奨）

**[TrailPaint を開く](https://notoriouslab.github.io/trailpaint/app/)**

リアルな地図の上に手描き風のルートイラストを作成。PWA 対応で、スマホのホーム画面にインストール可能。

### オフライン版

[`trailpaint.html`](trailpaint.html) を単一ファイルとしてダウンロードし、ブラウザで開くだけ。インターネット不要。

> ⚠️ iOS では「ファイル」アプリから直接ローカル HTML を開けません。オンライン版をご利用ください。

## これは何？

TrailPaint 路小繪は手描き風のトレイルマップ作成ツールです。スクリーンショットをアップロードするか、オンライン地図を使って、数分で美しい共有用イラストマップを作成できます。

## オンライン版の機能

### コア

- 🖊️ **ルート描画** — 手描き風点線 + 方向矢印、5色自動ループ
- 📍 **スポットカード** — 紙テクスチャの手描き風スタイル、写真・21種アイコン・ドラッグ配置
- 📷 **スクリーンショットベースマップ** — 画面に画像をドラッグすると自動切替、Google Maps スクリーンショット対応
- 🗺️ **マップスタイル切替** — 標準 / 衛星 / 等高線 / ダーク

### 出力

- 📐 **マルチ比率出力** — 1:1（IG）/ 9:16（Story）/ 4:3 / オリジナル
- 🖼️ **3種フレームスタイル** — クラシック二重枠 / 紙手描き / ミニマル細枠
- 📊 **統計オーバーレイ** — 距離・時間・獲得標高を自動で画像に印字
- 🔍 **高解像度** — 1x / 2x / 3x pixelRatio

### データ

- 📥 **GPX インポート** — トラック + ウェイポイントを解析、大量ポイントを自動簡略化
- ⛰️ **標高プロファイル** — Canvas 折れ線グラフ（距離 / 時間 / 獲得 / 損失）
- 🔭 **標高クエリ** — 手動で描いたルートも標高を取得可能（Open-Meteo API）
- 💾 **保存 / 読込** — JSON 形式、後方互換性あり

### 体験

- ↩️ **元に戻す / やり直し** — Cmd+Z キーボードショートカット
- 🌊 **手描き揺れ効果** — SVG フィルターエフェクト、ON/OFF 切替可
- 🌐 **3言語対応** — 中文 / English / 日本語、自動検出
- 📱 **PWA** — スマホのホーム画面にインストール可能
- 🎓 **ユーザーガイド** — 初回起動時の 3ステップチュートリアル + サンプルプロジェクト

## 使い方

```
1. 場所を検索する  または  スクリーンショットを画面にドラッグする
2.「スポット追加」をクリックして地図上にマーカーを配置、名前と写真を入力
3.「ルート描画」をクリックしてパスをなぞり、「ルート完成」をクリック
4. 出力比率とフレームスタイルを選択し、PNG をダウンロード
```

## 技術構成

### オンライン版（`online/`）

- **フレームワーク**: Vite + React 19 + TypeScript
- **地図**: Leaflet + react-leaflet（Carto Voyager タイル）
- **状態管理**: Zustand + zundo（temporal undo/redo）
- **PWA**: vite-plugin-pwa + Workbox
- **出力**: html-to-image + Canvas 後処理
- **i18n**: 軽量カスタム t() + ランタイムロケール検出
- **構造**: core/（Leaflet 非依存）+ map/（Leaflet 統合層）

### オフライン版（ルートディレクトリ）

- **フレームワーク**: Preact（React compat）+ 純粋なブラウザ API
- **レンダリング**: SVG オーバーレイ + Canvas 出力
- **単一ファイル**: すべての JS/CSS を HTML に内包

## 開発

```bash
# オンライン版
cd online
npm install
npm run dev        # 開発サーバー
npm run build      # ../app/ へビルド

# オフライン版（ビルド不要）
open trailpaint.html
```

## 既知の制限

- オンライン版は地図タイルの読み込みにインターネットが必要（PWA キャッシュ後はキャッシュ済みエリアをオフライン閲覧可）
- スクリーンショット ↔ 地図モードの切替でスポットとルートがクリアされる（座標系が異なるため）
- iOS ではローカル HTML ファイルを開けない（プラットフォーム制限 — オンライン版をご利用ください）
